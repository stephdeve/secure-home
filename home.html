02 · Fonctionnalités

Bien plus que 3 capteurs

Voici toutes les fonctionnalités à envisager pour avoir un produit complet, différenciant et difficile à copier rapidement.

🔥

Incendie

Capteur de fumée + thermomètre infrarouge. Confirmation par 2 capteurs avant alerte pour éviter les faux positifs.

MQ-135 DHT22 ESP32 → Pompiers
💨

Fuite de gaz

Détection méthane, CO, propane, butane. Seuils configurables par type de logement.

MQ-2 MQ-7 CO Arduino → Pompiers
🕵️

Intrusion

PIR + capteur magnétique portes/fenêtres + caméra IA de reconnaissance. Photo envoyée à la police.

HC-SR501 PIR ESP32-CAM → Police
💧

Inondation / dégât des eaux

Capteur d'humidité sol dans cave, cuisine, salle de bain. Alerte propriétaire + assureur optionnel.

Capteur HS-038 → Propriétaire
⚡

Coupure de courant / alarme secteur

Détection panne réseau. Le hub bascule automatiquement sur batterie de secours et alerte GSM.

UPS intégré Module GSM SIM800
🌡️

Qualité de l'air intérieur

Surveillance CO₂, humidité, température ambiante. Dashboard bien-être en prime — valeur ajoutée visible.

SCD30 CO₂ BME280
📹

Surveillance vidéo IA

Stream live sur mobile, détection silhouette, capture automatique lors d'alerte. Stockage cloud 7 jours.

ESP32-CAM IA Vision RTSP Stream
🔔

Alarme sonore locale

Sirène 90dB activée automatiquement. Peut dissuader un intrus avant même l'arrivée de la police.

Buzzer actif Sirène 12V
🤖

IA d'analyse comportementale

Apprentissage des patterns normaux du foyer. Alerte uniquement si comportement réellement anormal. Moins de fausses alertes.

ML embarqué TensorFlow Lite

07 · Risques & mitigations

Les obstacles à anticiper

Mieux vaut les identifier maintenant que de les découvrir en plein lancement.

HIG

Connexion légale aux autorités

En France/Afrique, alerter directement police/pompiers nécessite une habilitation CNAPS ou un partenariat avec un télé-opérateur agréé.

→ Solution : Partenaire avec un centre de télésurveillance agréé dès la phase 3.
HIG

Fausses alertes répétées

Un taux élevé détruit la confiance client ET épuise les services d'urgence. Risque de pénalités légales.

→ Solution : Double confirmation (2 capteurs + délai 30s) + apprentissage IA des patterns normaux.
MED

Coupure internet / courant

Le système devient inutile si la connexion tombe au mauvais moment.

→ Solution : Module GSM de secours (SIM800) + batterie LiPo 12h dans le hub.
MED

RGPD & vie privée

Caméras, données de comportement = données sensibles. Violation = amende + réputation.

→ Solution : Chiffrement E2E, stockage local optionnel, politique RGPD dès le prototype.
MED

Coûts de support & SAV

Le hardware en terrain = pannes, remplacements, techniciens. Sous-estimé par beaucoup de startups IoT.

→ Solution : Réseau d'installateurs partenaires locaux + diagnostic remote via plateforme.
LOW

Scalabilité technique

Gérer 1000+ hubs ESP32 simultanément avec MQTT demande une infrastructure solide.

→ Solution : MQTT Broker managé (HiveMQ Cloud) + architecture microservices dès le départ.
